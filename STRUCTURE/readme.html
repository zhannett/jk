Organise JS

This demo loads content into the page, using dust.js, a json document of content, using the data-behavior and javascript organisation as described in my .net magazine article. It needs to be run from a webserver, MAMP, etc.

The list below has a data attribute of data-behavior that has the value show_articles. The data-behavior attributes are looked for in the application.js and a method show_articles is triggered. show_articles loads in the dust template and loads in the json, it then passes these off to a function, m1312.Functions.generate_html() to generate the html. The results of this are returned to the show_articles behavior and are inserted into the page.

All the javascript is nicely organised, kept in small, readable chunks and is easy to pick through. There could be many templates and many data feeds, all of which only need one generic generate html function. The javascript here is also annotated so you can pick through easier.
